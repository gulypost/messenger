# Простой UDP-чат на C++ (Winsock2)

## Изменения в версии `win_v0.2.0`

По сравнению с прошлой версией `win_v0.1.0`:

* Убрано разделение на отдельные режимы `sender` и `listener`.
* Теперь программа одновременно отправляет и принимает сообщения.
* Для параллельной работы используются два потока:
  * один поток принимает входящие сообщения;
  * второй поток обрабатывает ввод пользователя и отправку сообщений.
* Вместо одного общего порта теперь указываются:
  * свой порт;
  * IP-адрес собеседника;
  * порт собеседника.
* Для завершения работы используется команда `/exit`.

---

Минималистичное консольное приложение для обмена сообщениями между двумя компьютерами под управлением Windows. Программа использует протокол **UDP**, что обеспечивает быструю передачу данных без необходимости постоянного соединения.

В новой версии чат работает в двух направлениях одновременно: пользователь может принимать входящие сообщения и сразу отправлять ответы, не переключаясь между режимами отправителя и слушателя.

## Особенности

* Работает по принципу двустороннего UDP-чата.
* Использует нативный Windows Sockets API (Winsock2).
* Поддерживает IPv4.
* Одновременно отправляет и принимает сообщения.
* Работает на любой версии Windows (x86/x64).
* Завершается командой `/exit`.

## Требования

* Операционная система: **Windows** (7, 8, 10, 11).
* Компилятор: **MinGW (g++)** или **MSVC (Visual Studio)**.

## Как собрать проект

### Способ 1: Через командную строку (MinGW/g++)

Для сборки необходимо явно указать линковку сетевой библиотеки `-lws2_32`:

```bash
g++ win_v0.2.0.cpp -o udp_chat.exe -lws2_32
```

### Способ 2: Visual Studio

1. Создайте новый проект "Консольное приложение C++".
2. Скопируйте код из `win_v0.2.0.cpp` в ваш проект.
3. Соберите решение. Библиотека подключится автоматически благодаря строке `#pragma comment` в коде.

## Как пользоваться

Программу нужно запустить на двух разных компьютерах или в двух разных окнах консоли для теста.

В отличие от прошлой версии, теперь не нужно выбирать режим `sender` или `listener`. Каждый запущенный экземпляр программы сразу умеет и отправлять, и принимать сообщения.

1. **На первом компьютере:**

   * Запустите `.exe`.
   * Введите свой порт, например `50005`.
   * Введите IP-адрес второго компьютера.
   * Введите порт второго компьютера, например `50006`.
   * После запуска можно вводить сообщения и получать ответы.

2. **На втором компьютере:**

   * Запустите `.exe`.
   * Введите свой порт, например `50006`.
   * Введите IP-адрес первого компьютера.
   * Введите порт первого компьютера, например `50005`.
   * После запуска можно вводить сообщения и получать ответы.

### Пример настройки

На первом компьютере:

```text
My port: 50005
Peer IP: 192.168.1.20
Peer port: 50006
```

На втором компьютере:

```text
My port: 50006
Peer IP: 192.168.1.10
Peer port: 50005
```

После этого оба пользователя могут писать сообщения в консоль и нажимать `Enter` для отправки.

### Выход из программы

Чтобы выйти из чата, введите:

```text
/exit
```

## Важные замечания

* **Брандмауэр:** при первом запуске Windows может заблокировать сетевую активность. Нужно разрешить доступ.
* **IP-адрес:** IP своего компьютера можно узнать командой `ipconfig`.
* **Сеть:** оба устройства должны находиться в одной сети или иметь доступ друг к другу по IP.
* **UDP:** протокол UDP не гарантирует доставку сообщений, поэтому при плохом соединении отдельные сообщения могут теряться.
