# Простой UDP-чат на C++ (Winsock2)

Минималистичное консольное приложение для обмена сообщениями между двумя компьютерами под управлением Windows. Программа использует протокол **UDP**, что обеспечивает быструю передачу данных без необходимости постоянного соединения.

## Особенности
*   Работает по принципу "Sender/Listener".
*   Использует нативный Windows Sockets API (Winsock2).
*   Поддерживает IPv4.
*   Работает на любой версии Windows (x86/x64).

## Требования
*   Операционная система: **Windows** (7, 8, 10, 11).
*   Компилятор: **MinGW (g++)** или **MSVC (Visual Studio)**.

## Как собрать проект

### Способ 1: Через командную строку (MinGW/g++)
Для сборки необходимо явно указать линковку сетевой библиотеки `-lws2_32`:
```bash
g++ win_v0.1.0.cpp -o udp_chat.exe -lws2_32
```

### Способ 2: Visual Studio
1. Создайте новый проект "Консольное приложение C++".
2. Скопируйте код из `win_v0.1.0.cpp` в ваш проект.
3. Соберите решение (Build Solution). Библиотека подключится автоматически благодаря строке `#pragma comment` в коде.

## Как пользоваться

Программу нужно запустить на двух разных компьютерах (или в двух разных окнах консоли для теста).

1.  **На первом компьютере (Слушатель/Listener):**
    *   Запустите `.exe`.
    *   Введите любой IP (в режиме слушателя он не используется для бинда, но программа запрашивает его в начале).
    *   Введите порт (например, `50005`).
    *   Выберите режим `2` (listener).
    *   *Программа перейдет в режим ожидания сообщений.*

2.  **На втором компьютере (Отправитель/Sender):**
    *   Запустите `.exe`.
    *   Введите **IP-адрес первого компьютера**.
    *   Введите тот же порт (`50005`).
    *   Выберите режим `1` (sender).
    *   *Теперь можно вводить текст и нажимать Enter для отправки.*

### Важные замечания
*   **Брандмауэр:** При первом запуске Windows может заблокировать сетевую активность.
*   **Выход:** Чтобы выйти из режима отправителя, введите команду `/exit`.
